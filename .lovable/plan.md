
# SAME AGE — React版再構築プラン

## 概要
アップロードされたバニラHTML/JS/CSSの「SAME AGE」アプリをReact + TypeScript + Tailwind CSSで再構築し、Firebaseをバックエンドとして維持します。

## ログイン問題の修正
- Firebaseコンソールで **Lovableのプレビュードメイン** (`*.lovable.app`) を認証済みドメインに追加する必要あり（ユーザー側の作業）
- `signInWithPopup` のエラーハンドリングを強化し、`signInWithRedirect` へのフォールバックを改善
- Firestoreアクセス失敗時も画面遷移が止まらないよう修正

## 画面構成（全9画面）
1. **イントロ画面** — ロゴ・タグライン・「はじめる」ボタン
2. **ログイン画面** — Googleログインボタン
3. **ロール選択画面** — 父親 or 子供の選択
4. **ペアリング画面** — 家族コードの作成・参加
5. **セットアップ画面** — 年齢入力（子の年齢・父の年齢・父の名前）
6. **カード体験画面** — 5枚の質問カード（DREAM/FEAR/SECRET/JOY/REGRET）
7. **回答入力画面** — テキスト入力＋音声入力対応
8. **エンディング画面** — 体験完了メッセージ
9. **タイムカプセル画面** — 過去の回答振り返り

## 主要機能
- **Firebase Auth** でGoogleログイン（既存設定を維持）
- **Firestore** で家族データ・回答の保存
- **家族ペアリング** — 6桁コードで父子をマッチング
- **カードスワイプUI** — 質問カードのめくり体験
- **レスポンシブ対応** — モバイルファーストのデザイン
- アップロードされたCSSのデザインテーマ（クリーム色・和風テイスト）を再現

## デザイン
- 既存のカラーパレット（cream, amber, brown, sage）をTailwind CSS変数として移植
- Shippori Mincho + Zen Kaku Gothic New フォントを維持
- パーティクルアニメーション・グレインテクスチャを再現

## 技術構成
- React Router でページ遷移管理
- Firebase SDK（npm版）を追加
- Context APIで認証状態・家族データを管理
- 環境変数でFirebase設定を管理
